# 🚀 Northflank 零成本双子星节点部署方案 (混合双打增强版)

本项目是一个专为 Northflank 容器云环境深度定制的 VLESS 节点部署方案。基于 **Sing-box** 与 **Cloudflared** 官方核心，采用极简的 Alpine 底层与多阶段构建技术，为您打造极高纯净度、长期稳定且防扫防封的专属网络通道。

## ✨ 核心特性

* **🐳 极致轻量纯净**：源码流 `Dockerfile` 多阶段构建，摒弃臃肿的第三方镜像，内存占用极低，完美适配 Northflank 免费层 (256MB) 规格。
* **⚔️ 混合双打架构**：
  * **保底防封 (WS 协议)**：运行于内部 `8080` 端口，通过 Cloudflare Argo 隧道内网穿透直达，完全关闭公网入口，实现物理级隐蔽。
  * **极速原生 (gRPC 协议)**：运行于 `8081` 端口，直接利用 Northflank 官方强大的 HTTP/2 网关直连，提供极低延迟的丝滑体验。
* **🤖 智能直出配置**：抛弃繁琐的手动拼接。部署成功后，系统将在容器日志 (Logs) 中自动组装并打印包含您专属参数的客户端节点链接，一键复制即可使用。
* **🛡️ 零接触风控**：配置参数 (UUID、域名、Token) 均通过云端环境变量动态注入，无需在代码中硬编码，彻底保障您的账号与隐私安全。

1. **GitHub 账号**：用于 Fork 或存储本项目的代码。

3. **Cloudflare 账号**：拥有一个托管在 CF 的域名，并开通 Zero Trust（免费）。

## 🚀 极简部署指南

### 第一步：配置 Cloudflare 隧道 (为 WS 节点提速)
1. 登录 Cloudflare Zero Trust，导航至 **Networks -> Tunnels**。
2. 创建一个 Cloudflared 隧道，复制生成的 `ey...` 开头的 **Token**。
3. 在隧道的 **Public Hostname** 中添加一条规则：
   - 绑定你想用的加速域名（例如：`ws.yourdomain.com`）。
   - Service 配置为：`HTTP://localhost:8080`。

### 第二步：Northflank 平台部署
1. 登录 Northflank，在您的 Project 中新建一个 **Combined Service (联合服务)**。
2. 绑定您存放此代码的 GitHub 仓库，构建方式保持默认的 `Dockerfile`。
3. **开放直连端口 (为 gRPC 准备)**：
   - 在左侧 **Networking (网络)** -> **Ports** 设置中，点击 Add port。
   - 端口填 `8081`，协议必须选择 `HTTP/2`。
   - 勾选 `Publicly expose this port to the internet`，并保存配置。
   - 复制生成的绿色公网域名（例如：`p01--xxx.code.run`）。
4. **注入环境变量**：
   在左侧 **Environment (环境变量)** 中添加以下四个关键变量：
   - `UUID`：您的专属 VLESS UUID（例：`c81e134a...`）。
   - `ARGO_TOKEN`：第一步中获取的 CF 隧道 Token。
   - `ARGO_DOMAIN`：第一步中绑定的 CF 隧道域名（例：`ws.yourdomain.com`）。
   - `NF_DOMAIN_GRPC`：第三步中复制的 Northflank 官方直连域名（例：`p01--xxx.code.run`）。

### 第三步：一键获取节点
点击页面右上角的 **Deploy / Restart**。等待 1-2 分钟镜像构建完成并启动后，点击进入 **Logs (日志)** 页面。您将看到自动生成的双协议节点链接，直接复制导入客户端（如 v2rayN / Clash / sing-box）即可起飞！
